# Mitwirken / Contributing

Danke für dein Interesse an NetzPilot!

## Lizenzmodell — bitte vor einem Beitrag lesen

NetzPilot ist **dual lizenziert** (GNU AGPL-3.0 + kommerzielle Lizenz, siehe [LICENSING.md](LICENSING.md)). Damit der Rechteinhaber das Projekt weiterhin **sowohl** quelloffen (AGPL) **als auch** kommerziell lizenzieren kann, gelten für Beiträge die folgenden Bedingungen.

**Mit dem Einreichen eines Beitrags (z. B. Pull Request) erklärst du dich damit einverstanden, dass:**

1. dein Beitrag unter der **GNU AGPL-3.0** lizenziert wird, **und**
2. du dem Rechteinhaber (**Amar Akram**) ein unwiderrufliches, weltweites, gebührenfreies Recht einräumst, deinen Beitrag **auch unter anderen Lizenzbedingungen (einschließlich einer kommerziellen Lizenz)** zu nutzen, zu lizenzieren und weiterzugeben, **und**
3. du bestätigst, dass du berechtigt bist, den Beitrag unter diesen Bedingungen einzureichen (vgl. Developer Certificate of Origin, DCO 1.1).

## Sign-off (DCO)

Bestätige Punkt 3, indem du jede Commit-Nachricht mit einem `Signed-off-by` versiehst:

```bash
git commit -s -m "kurze Beschreibung"
```

Das fügt eine Zeile `Signed-off-by: Dein Name <deine@email>` hinzu — gemäß dem [Developer Certificate of Origin](https://developercertificate.org/).

## Praktisches

- Stil und Konventionen am bestehenden Code orientieren (siehe `CLAUDE.md`/`AGENTS.md` sind nicht Teil des öffentlichen Repos — halte dich an den vorhandenen Code).
- Tests grün halten: `python scripts/run_all_checks.py`.
- Für größere Änderungen bitte vorab ein Issue öffnen.
